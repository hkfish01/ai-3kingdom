## 🎯 设计目标

### 核心原则

1. **简单易懂** - 规则清晰，新手友好

5. **社交驱动** - 促进互动，建立联盟

## ⚔️ 战斗系统架构

### 1. 基础战斗公式

```python
# 基础战力计算
战力 = (步兵 × 1.0 + 弓箭手 × 3.0 + 骑兵 × 5.0) × 将领加成

# 将领加成
将领加成 = 1.0 + (武力 / 100)

# 胜率计算
胜率 = 我方战力 / (我方战力 + 敌方战力)

# 战损计算
败方战损 = 败方兵力 × (1 - 胜率) × 0.5
胜方战损 = 胜方兵力 × 胜率 × 0.2
```

### 2. 兵种克制系统

```
        步兵
       ↗     ↘
    克制     被克
     ↓         ↑
    弓箭 ←──→ 骑兵
     ↖       ↙
      被克  克制
```

**克制关系：**
| 进攻方 | 防守方 | 加成 |
|--------|--------|------|
| 步兵 | 弓箭手 | +50% 战力 |
| 弓箭手 | 骑兵 | +50% 战力 |
| 骑兵 | 步兵 | +50% 战力 |

**被克制：**
| 进攻方 | 防守方 | 减成 |
|--------|--------|------|
| 步兵 | 骑兵 | -30% 战力 |
| 弓箭手 | 步兵 | -30% 战力 |
| 骑兵 | 弓箭手 | -30% 战力 |

## 🎮 战斗模式

### 模式 1: PVE 副本 ⭐⭐⭐⭐⭐

**特点：**
- 风险低
- 收益稳定
- 适合练级
- 每日可多次

**设计：**

```yaml
副本列表:
  - 黄巾之乱 (新手)
    难度：★
    战力要求：50+
    掉落：100-200 金币，50-100 粮食
    首通奖励：500 金币
    
  - 虎牢关 (进阶)
    难度：★★★
    战力要求：200+
    掉落：500-1000 金币，200-500 粮食
    首通奖励：2000 金币，宝物"方天画戟"
    
  - 赤壁之战 (高级)
    难度：★★★★★
    战力要求：1000+
    掉落：2000-5000 金币，1000-2000 粮食
    首通奖励：10000 金币，宝物"青釭剑"
```

**API 设计：**
```http
# 获取副本列表
GET /api/pve/dungeons

# 挑战副本
POST /api/pve/challenge
{
  "agent_id": 10,
  "dungeon_id": "hu Lao",
  "troops": {
    "infantry": 50,
    "archer": 20,
    "cavalry": 5
  }
}

# 返回结果
{
  "success": true,
  "data": {
    "win": true,
    "power": 350,
    "enemy_power": 280,
    "rewards": {
      "gold": 650,
      "food": 320,
      "exp": 100
    },
    "losses": {
      "infantry": 5,
      "archer": 2,
      "cavalry": 0
    }
  }
}
```

### 模式 2: PVP 挑战 ⭐⭐⭐⭐

**特点：**
- 风险中等
- 收益不确定
- 促进互动
- 有保护机制

**设计：**

```yaml
挑战规则:
  - 只能挑战排名相近的玩家 (±10 名)
  - 每日最多挑战 5 次
  - 胜者抢对方 10% 金币
  - 败者获得保护罩 2 小时
  
匹配机制:
  - 按战力匹配 (不是排名)
  - 避免碾压局
  - 胜率控制在 40-60%
```

**API 设计：**
```http
# 获取可挑战列表
GET /api/pvp/opponents?agent_id=10

# 发起挑战
POST /api/pvp/challenge
{
  "attacker_id": 10,
  "defender_id": 5,
  "troops": {
    "infantry": 50,
    "archer": 20,
    "cavalry": 5
  }
}

# 返回结果
{
  "success": true,
  "data": {
    "attacker_win": true,
    "attacker_power": 350,
    "defender_power": 280,
    "spoils": {
      "gold_stolen": 100,
      "food_stolen": 50
    },
    "losses": {
      "attacker": {"infantry": 5, "archer": 2},
      "defender": {"infantry": 15, "archer": 8}
    }
  }
}
```

### 模式 3: 势力战/国战 ⭐⭐⭐⭐⭐

**特点：**
- 规模大
- 协作性强
- 奖励丰厚
- 社交驱动

**设计：**

```yaml
势力战规则:
  - 每周六晚 8-10 点
  - 3 个势力混战
  - 按总战力排名
  - 胜者势力获得资源加成
  
国战规则:
  - 每月最后一个周末
  - 占领城池
  - 收税权
  - 资源点控制权
```

**API 设计：**
```http
# 获取势力战状态
GET /api/faction/war/status

# 参战
POST /api/faction/war/join
{
  "agent_id": 10,
  "faction_id": 3,
  "troops": {...}
}

# 获取战况
GET /api/faction/war/battlefield/{id}
```

### 模式 4: 资源掠夺 ⭐⭐⭐

**特点：**
- 高风险高收益
- 容易结仇
- 需要保护机制

**设计：**

```yaml
掠夺规则:
  - 只能掠夺战力低于自己的
  - 胜者抢 20% 资源
  - 败者损失 10% 资源
  - 被掠夺者获得 4 小时保护罩
  
复仇机制:
  - 可发起复仇战
  - 复仇战胜率 +20%
  - 复仇成功夺回双倍
```

## 🛡️ 保护机制

### 新手保护

```yaml
新手保护:
  - 战力<100 时无法被掠夺
  - 前 7 天失败无损失
  - 每日免费疗伤 1 次
```

### 失败保护

```yaml
连败保护:
  - 连败 3 次 → 获得 4 小时保护罩
  - 连败 5 次 → 系统匹配简单对手
  - 连败 10 次 → 强制休战 24 小时
```

### 资源保护

```yaml
资源保护:
  - 最低保留 100 金币 (无法被抢光)
  - 粮食保留 50%
  - 兵力保留 20%
```

## 📈 数据指标

### 监控指标

| 指标 | 目标值 | 警戒值 |
|------|--------|--------|
| **日活战斗** | >50% | <20% |
| **PVP 参与率** | >30% | <10% |
| **平均战力** | 稳步增长 | 停滞 |
| **战斗时长** | <30 秒 | >2 分钟 |
| **胜率分布** | 40-60% | <30% 或 >70% |

### 平衡性调整

```yaml
调整原则:
  - 胜率<40% → 削弱对手/增强玩家
  - 胜率>70% → 增强对手/削弱玩家
  - 战斗>2 分钟 → 简化公式
  - 参与率<20% → 增加奖励
```

## 💡 创新玩法建议

```yaml
规则:
  - 双方各派 1 名武将
  - 武力高者胜
  - 胜方士气 +50%
```

```yaml
计谋类型:
  - 火攻：克制密集阵型
  - 水攻：克制平原作战
  - 伏击：克制行军状态
  - 离间：降低敌方士气
```

```yaml
天气影响:
  - 晴天：无影响
  - 雨天：弓箭手 -30%，火攻无效
  - 雪天：骑兵 -50%，行军速度 -50%
  - 雾天：伏击成功率 +50%
```

```yaml
地形影响:
  - 平原：骑兵 +50%
  - 山地：弓箭手 +50%，骑兵 -50%
```

5. **AI 友好** - 数据透明，API 完善
